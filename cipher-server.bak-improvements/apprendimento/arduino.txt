# Arduino – Guida Completa e Moduli

---

## Cos'è Arduino

Arduino è una piattaforma open-source per prototipazione elettronica, composta da hardware (microcontrollori) e software (IDE). Creata nel 2005 da Massimo Banzi e colleghi al IDII di Ivrea (Italia). È la piattaforma più diffusa al mondo per l'elettronica hobbistica, educativa e professionale. Programmatoin C/C++.

---

## Differenza Arduino vs Raspberry Pi

| Aspetto          | Arduino                    | Raspberry Pi               |
|------------------|----------------------------|----------------------------|
| Tipo             | Microcontrollore           | Single-Board Computer      |
| OS               | No (bare metal)            | Linux                      |
| Linguaggio       | C/C++ (subset)             | Python, C, molti altri     |
| GPIO             | 5V (tollerante)            | 3.3V (sensibile)           |
| ADC              | Sì (integrato)             | No (solo digitale)         |
| Real-time        | Sì (deterministico)        | No (OS introduce latenza)  |
| Consumo          | Molto basso (mA)           | Alto (centinaia mA)        |
| Avvio            | Istantaneo                 | 30-60 secondi              |
| Uso tipico       | Controllo hardware         | Elaborazione, networking   |

## Modelli Arduino

### Arduino Uno (R3)
- MCU: ATmega328P 16MHz
- Flash: 32KB, SRAM: 2KB, EEPROM: 1KB
- Digital I/O: 14 (6 PWM)
- Analog Input: 6
- Alimentazione: USB o 7-12V jack
- Il modello più usato per imparare

### Arduino Uno R4
- **Minima**: MCU Renesas RA4M1 48MHz, più potente
- **WiFi**: + modulo ESP32-S3 per WiFi/BT + LED matrix 12x8

### Arduino Mega 2560
- MCU: ATmega2560 16MHz
- Flash: 256KB, SRAM: 8KB
- Digital I/O: 54 (15 PWM)
- Analog Input: 16
- Più pin e memoria per progetti complessi

### Arduino Nano
- Stesso MCU dell'Uno (ATmega328P)
- Formato mini, per breadboard
- Varianti: Nano Every (ATmega4809), Nano 33 IoT (WiFi/BT)

### Arduino Nano 33 BLE / BLE Sense
- MCU: nRF52840 64MHz
- BLE integrato
- **BLE Sense**: + IMU, microfono, sensori ambientali, gesture
- Usato per TinyML (machine learning su microcontrollori)

### Arduino Leonardo
- MCU: ATmega32U4
- USB nativo: può emulare tastiera/mouse (HID)

### Arduino Micro
- Come Leonardo in formato ridotto

### Arduino Due
- MCU: AT91SAM3X8E ARM Cortex-M3 84MHz
- Flash: 512KB, SRAM: 96KB
- 3.3V logica (attenzione!)
- Molto più potente degli AVR

### Arduino MKR Series
- Famiglia per IoT, tutti a 3.3V
- **MKR WiFi 1010**: WiFi + BT (ESP32)
- **MKR NB 1500**: NB-IoT / LTE-M
- **MKR GSM 1400**: GSM/3G
- **MKR WAN 1300**: LoRa
- **MKR Zero**: SD card + I²S audio

### Arduino Portenta H7
- MCU: STM32H747 dual-core Cortex-M7 + M4 480MHz
- Per applicazioni industriali/professionali
- Supporta MicroPython, OpenMV

### Arduino Giga R1 WiFi
- MCU: STM32H747 (come Portenta)
- Grande formato (come Mega)
- WiFi/BT, display connector, microfono

## IDE e Programmazione

### Arduino IDE 2
Software ufficiale gratuito per Windows, macOS, Linux.
- Editor con autocompletamento
- Serial Monitor e Plotter integrati
- Librerie manager integrato
- Debugger (per schede supportate)

### PlatformIO
IDE avanzato (plugin VS Code).
- Supporta centinaia di piattaforme
- Build system potente
- Test unitari

### Struttura sketch base
```cpp
// Variabili globali
int ledPin = 13;

void setup() {
    // Eseguito una volta all'avvio
    pinMode(ledPin, OUTPUT);
    Serial.begin(9600);
    Serial.println("Arduino avviato!");
}

void loop() {
    // Eseguito in loop continuo
    digitalWrite(ledPin, HIGH);
    delay(1000);
    digitalWrite(ledPin, LOW);
    delay(1000);
}
```

## Funzioni fondamentali

### GPIO digitale
```cpp
pinMode(pin, OUTPUT);           // configura come output
pinMode(pin, INPUT);            // input senza pull
pinMode(pin, INPUT_PULLUP);     // input con pull-up interno

digitalWrite(pin, HIGH);        // metti a 5V
digitalWrite(pin, LOW);         // metti a 0V
int val = digitalRead(pin);     // leggi 0 o 1
```

### PWM (analogico simulato)
```cpp
// Solo pin PWM (3,5,6,9,10,11 su Uno)
analogWrite(pin, valore);       // 0-255 (0-100% duty cycle)
// 0 = 0%, 127 = 50%, 255 = 100%
```

### ADC (lettura analogica)
```cpp
int val = analogRead(A0);       // 0-1023 (10-bit, 0-5V)
float tensione = val * (5.0 / 1023.0);
```

### Tempo
```cpp
delay(1000);                    // aspetta 1000ms (blocca tutto)
delayMicroseconds(100);         // aspetta 100µs
unsigned long t = millis();     // ms dall'avvio
unsigned long t = micros();     // µs dall'avvio
```

### Seriale
```cpp
Serial.begin(9600);
Serial.print("Valore: ");
Serial.println(valore);
Serial.print(valore, DEC);      // decimale
Serial.print(valore, HEX);      // esadecimale
Serial.print(valore, BIN);      // binario
Serial.available();             // bytes disponibili
char c = Serial.read();
String s = Serial.readString();
```

### EEPROM
```cpp
#include <EEPROM.h>
EEPROM.write(indirizzo, valore);    // 0-255
int val = EEPROM.read(indirizzo);
EEPROM.put(indirizzo, variabile);   // qualsiasi tipo
EEPROM.get(indirizzo, variabile);
```

## Comunicazione

### I²C (Wire)
```cpp
#include <Wire.h>

Wire.begin();                           // master
Wire.beginTransmission(0x48);          // indirizzo slave
Wire.write(0x00);                       // registro
Wire.endTransmission();

Wire.requestFrom(0x48, 2);             // richiedi 2 byte
while (Wire.available()) {
    byte b = Wire.read();
}
```

### SPI
```cpp
#include <SPI.h>

SPI.begin();
digitalWrite(SS, LOW);
byte response = SPI.transfer(0x01);
digitalWrite(SS, HIGH);
```

### SoftwareSerial (UART aggiuntivo)
```cpp
#include <SoftwareSerial.h>
SoftwareSerial mySerial(10, 11);    // RX, TX
mySerial.begin(9600);
mySerial.println("Ciao");
```

## Shield (espansioni)

### Ethernet Shield (W5100/W5500)
- Aggiunge connettività Ethernet
- Stack TCP/IP hardware
- Include slot SD card

### WiFi Shield
- ESP8266 o ESP32 come coprocessore WiFi
- Comunicazione via UART con comandi AT

### Motor Shield (L293D / L298N)
- **Adafruit Motor Shield V2**: 4 motori DC o 2 stepper
- Usa I²C (PCA9685 per PWM)
- **L298N Module**: 2 motori DC, driver H-bridge

### Relay Shield
- 2, 4 o 8 relè
- Controllo carichi AC (luci, motori, elettrodomestici)
- Isolamento ottico

### Sensor Shield
- Aggiunge pin con GND, VCC e segnale per ogni GPIO
- Facilita collegamento sensori senza breadboard

### SD Card Shield
- Slot SD per storage
- Interfaccia SPI

### GSM/GPRS Shield (SIM900/SIM800)
- Connettività cellulare 2G/3G
- SMS, chiamate, dati
- Antenna esterna

### GPS Shield
- Modulo GPS integrato (NEO-6M)
- UART per dati NMEA

### Bluetooth Shield (HC-05)
- Bluetooth seriale 2.1
- Comunicazione con smartphone

### Prototyping Shield
- Area breadboard piccola sopra Arduino
- Perfetta per prototipi compatti

### Joystick Shield
- Joystick analogico + bottoni
- Per controller e giochi

### OLED Shield (Wemos D1)
- Display OLED integrato
- Per Wemos D1 Mini (ESP8266)

- **LDR**: fotoresistore
- **TCS3200 / TCS34725**: sensore colore
- **MAX30100 / MAX30102**: ossimetro + battito cardiaco

### Display e output
- **LCD 16x2 / 20x4**: alfanumerico con HD44780

- **OLED 0.96" SSD1306**: 128x64 I²C

- **TFT 1.8" ST7735**: 128x160 colori SPI

### Motori e attuatori

## Ecosistema ESP8266 / ESP32

Molto usati con o al posto di Arduino per progetti IoT.

### ESP8266 (NodeMCU / Wemos D1 Mini)
- WiFi 802.11 b/g/n integrato
- CPU: Tensilica L106 80/160MHz
- Flash: 1-4MB
- GPIO: 11 (molti con funzioni multiple)
- ADC: 1 (0-1V, non 0-5V!)
- Programmabile con Arduino IDE

### ESP32 / ESP32-S2 / ESP32-S3 / ESP32-C3
- WiFi + Bluetooth integrati
- CPU: dual-core Xtensa LX6 240MHz (ESP32 classico)
- ADC: 18 canali 12-bit
- DAC: 2 canali 8-bit
- Touch: 10 pin capacitivi
- Hall sensor, temperatura interna
- Ultra low power modes
- Supporta Arduino IDE, MicroPython, ESP-IDF

```cpp
// ESP32 con Arduino IDE
#include <WiFi.h>
WiFi.begin("SSID", "password");
while (WiFi.status() != WL_CONNECTED) delay(500);
Serial.println(WiFi.localIP());
```

## Best practices Arduino

- Usare `millis()` invece di `delay()` per multitasking
- Non usare `String` (frammentazione heap) — usare char array
- Controllare sempre la tensione logica dei moduli (3.3V vs 5V)
- Aggiungere condensatori di bypass (100nF) vicino ai IC
- Usare resistori pull-up/pull-down sui pin floating
- Calibrare i sensori analogici
- Proteggere i pin da sovratensioni con resistori serie
- Usare `const` e `constexpr` invece di `#define` dove possibile
- Commentare il codice con unità di misura e pin usati
